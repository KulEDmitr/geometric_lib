## ___Цели и задачи тестирования___

- Убедиться в корректной работе функций вычисления площади и периметра (или, в случае треугольника, периметра и площади по стороне и высоте) при различных наборах входных значений.

## ___Основные задачи___

- Проверить корректность вычисления геометрических параметров при положительных входных данных.
- Убедиться в корректной обработке отрицательных значений, когда в коде явно возвращаются специальные коды ошибок (например, -1, -2, -3).
- Протестировать большие значения аргументов (проверка на переполнение или корректность расчётов).

## ___Описание тестируемого продукта___

### ___circle.py___ 

`area(r)`: вычисляет площадь круга, возвращает `π * r * r`, при отрицательном `r`– возвращает `-1`.

`perimeter(r)`: вычисляет периметр (окружность) круга, возвращает `2 * π * r`, при отрицательном `r` – возвращает `-1`.

### ___rectangle.py___

`area(a, b)`: вычисляет площадь прямоугольника `(a * b)`, при отрицательном `a` возвращает `-1`, при отрицательном `b` – `-2`.

`perimeter(a, b)`: вычисляет периметр прямоугольника `(2 * (a + b))`, при отрицательном `a` возвращает `-1`, при отрицательном `b` – `-2`.

### ___square.py___

`area(a)`: вычисляет площадь квадрата `(a * a)`, при отрицательном `a` – возвращает `-1`.

`perimeter(a)`: вычисляет периметр квадрата `(4 * a)`, при отрицательном `a` – возвращает `-1`.

### ___triangle.py___

`area(a, h)`: вычисляет площадь треугольника `(a * h / 2)`, при отрицательном `a` – возвращает `-1`, при отрицательном `h` – `-2`.

`perimeter(a, b, c)`: вычисляет периметр треугольника `(a + b + c)`, при отрицательном `a` – возвращает `-1`, при отрицательном `b` – `-2`, при отрицательном `c` – `-3`.


## ___Область тестирования___
- *Корректные положительные значения: проверка правильности вычислений.*
- *Отрицательные значения: проверка корректности возвращаемого кода ошибки.*
- *Очень большие значения: проверка вычислений при больших входных данных (например, 1_000_000_000 и выше).*


## ___Стратегия тестирования___
- Тип тестирования: модульное (unit) тестирование.
- Техники тестирования:
- - Позитивные тесты: тестирование с корректными положительными аргументами.
- - Негативные тесты: тестирование с отрицательными входными данными, на которые код отвечает специальным значением (-1, -2 и т. д.).
- - Граничные/большие значения: тестирование для проверки корректности работы при экстремально больших числах.
Инструменты: стандартный модуль unittest в Python.


## ___Критерии приёмки___
- Все тесты на корректные (положительные) входные данные должны возвращать точные значения, соответствующие математическим формулам.
- При отрицательных значениях соответствующие функции должны возвращать код ошибки (-1, -2, -3) согласно описанию.
- Тесты с большими числами должны корректно выполняться (возвращать верные математические результаты или, при невозможности, соответствующие значения ошибок).
- Все тесты должны успешно выполняться в среде unittest (статус “OK”).


## __Ожидаемые результаты__
- Формируется подробный отчёт о прохождении тестов: все тесты должны выдать результат `“OK”`, если функции работают правильно в соответствии со спецификацией.

- В случае выявления расхождений (ошибок в расчёте либо невыполнения условий возвращаемого кода при отрицательных входных данных) будет получен `“FAIL”`.
Для каждого теста ожидается либо корректное численное значение (для позитивных тестов), либо корректный код ошибки (для негативных).
