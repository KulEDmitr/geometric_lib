# Unit-тестирование модулей библиотеки
Для обеспечения надежности и гарантии корректной работы планируется провести детальное unit-тестирование каждого модуля с использованием случайных тестов для имитации взаимодействия с конечным потребителем. 

## Требования
Необходимо протестировать корректность и безопасность функций в соответствующих модулях. Нужно учесть максимальное количество разных сценариев их использования, даже самые непредсказуемые:
* `circle.area` - вычисляет площадь окружности по радиусу
* `circle.perimeter` - вычисляет периметр окружности по радиусу
* `square.area` - вычисляет площадь квадрата по стороне
* `square.perimeter` - вычисляет периметр квадрата по стороне
* `rectangle.area` - вычисляет площадь прямоугольника по 2-м сторонам
* `rectangle.perimeter` - вычисляет периметр прямоугольника по 2-м сторонам
* `triangle.area` - вычисляет площадь прямоугольного треугольника по стороне и высоте
* `triangle.perimeter` - вычисляет периметр треугольника по 3-м сторонам

## Критерий успешного прохождения тестов
Тест считается пройденным в единственном случае: результат работы функции соответствует ожиданию. Результат проверяется на соответствие с помощью встроенной в `Python` библиотеки `unittest`

* `+ri` - *случайное натуральное число*

* `rs` - *случайная строка произвольной длины, состоящая из латинских букв*
* `rn` - *случайное целое число в строковом представлении*
