# Общее описание решение
**В функцию подаются значения, зависящие от параметров фигуры. Эти значения обрабатываются согласно математическим формулам указаным ниже. Функция возвращает площадь и периметр заданной фигуры.**

# Math formulas
## Area
- **Circle:** S = πR²
- **Rectangle:** S = ab
- **Square:** S = a²

## Perimeter
- **Circle:** P = 2πR
- **Rectangle:** P = 2a + 2b
- **Square:** P = 4a

## square area
 Функция принимает значение длины стороны квадрата и возвращает площадь квадрата.

 **Пример вызова**
```
   Параметры:
          a=2 

   Возвращаемое значение:
          4
```


## square perimeter

**Пример вызова**

# Отчет о тестировании

**Цели и задачи тестирования:**
Основная цель тестирования - проверка корректности функций для вычисления площади и периметра геометрических фигур (квадрат, прямоугольник, круг, треугольник). Задачи включают в себя тестирование всех функций на стандартные и крайние случаи, а также проверка обработки некорректных значений.

**Описание тестируемого продукта:**  
Продукт — это модуль, реализующий функции для вычисления площади и периметра различных геометрических фигур. Включает функции для квадрата, прямоугольника, круга и треугольника. Требования к продукту: каждая функция должна корректно обрабатывать положительные, нулевые и отрицательные значения.

**Область тестирования:**
В тестировании будут проверены функции: 
- square.area
- square.perimeter
- rectangle.area
- rectangle.perimeter
- circle.area
- circle.perimeter  
- triangle.area
- triangle.perimeter 

**Стратегия тестирования:** 
Использование юнит-тестирования с библиотекой unittest. Методы тестирования включают функциональное тестирование для проверки корректности выполнения вычислений и обработки граничных случаев (нулевые и отрицательные значения).

**Критерии приемки:**  
Продукт будет считаться успешно протестированным, если все тесты пройдут успешно, т.е.:
- Все функции возвращают ожидаемые результаты для стандартных и крайних случаев.
- Функции корректно обрабатывают отрицательные значения.
- Не возникает исключений и ошибок при выполнении тестов.

**Ожидаемые результаты:**  
Ожидаемые результаты тестирования включают:
- Все тесты должны пройти успешно без ошибок.
- Отчеты о дефектах не должны быть сгенерированы.
- Статус тестирования: "OK" при выполнении всех условий критериев приемки.

**Краткое заключение:**
Тестирование функций, реализующих вычисления для геометрических фигур, планируется провести с полной проверкой всех возможных ситуаций, включая негативные сценарии. Результаты тестирования будут использованы для окончательной оценки качества реализованного продукта.
